## 1. 项目概述

### 1.1 项目简介

Robit 是一个基于 Rust 的 AI 编程代理框架，采用单体仓库（Monorepo）架构设计。它提供：

- 统一的多提供商 LLM API 层
- 事件驱动的 Agent 运行时
- 可插拔的前端抽象（Terminal UI、Desktop GUI）
- 可扩展的工具系统和技能系统
- 跨平台支持（Windows、Linux、macOS）

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **模块化** | 清晰的 crate 划分，职责单一，依赖单向 |
| **可扩展** | 通过 trait 定义抽象，方便添加新前端、新工具 |
| **异步优先** | 基于 tokio 异步运行时，流式处理 |
| **容错设计** | 区分工具错误和系统错误，自动重试策略 |
| **配置驱动** | 统一配置文件，支持项目和全局两级 |

### 1.3 技术栈

| 领域 | 技术选型 | 版本 |
|------|----------|------|
| 编程语言 | Rust | 稳定版 |
| 异步运行时 | tokio | 1.x |
| LLM API | async-openai | 0.41 |
| 序列化 | serde + serde_json + toml | 1.x |
| 错误处理 | thiserror + anyhow | 2.x / 1.x |
| TUI 框架 | ratatui + crossterm | 0.30 / 0.29 |
| GUI 框架 | Tauri v2 + React | 2.x |
| 数据库 | SQLite + rusqlite | 0.40 (bundled) |
| CLI 解析 | clap | 4.5 |

#### 3.1.4 重试策略

| 错误类型 | 重试次数 | 退避策略 |
|----------|----------|----------|
| 连接超时 | 3 次 | 指数退避 (1s → 2s → 4s) |
| 速率限制 (429) | 3 次 | 尊重 Retry-After 头 |
| 服务端错误 (5xx) | 2 次 | 指数退避 |
| 流式中断 | 1 次 | 立即重试 |
| 认证失败 | 0 次 | 需要用户干预 |
| 客户端错误 (4xx) | 0 次 | 请求本身问题 |

```rust

    pub initial_backoff_ms: u64,    // 默认 1000
    pub max_backoff_ms: u64,        // 默认 30000
}
```

#### 3.2.2 Agent 主循环

```
用户输入 (FrontendMessage::UserInput)
    │
    ├─→ 检查技能触发 (SkillRegistry::match_trigger)
    │       └─→ 是 → 注入技能内容 → run_skill_turn()
    │
    └─→ 否 → run_turn()
            │
            └─→ 循环 run_one_step() (最多 20 次)
                    │
                    ├─→ 上下文截断 (ContextManager)
                    ├─→ 调用 LLM (流式)
                    │       ├─→ 发射 TextDelta 事件
                    │       └─→ 收集 ToolCall 片段
                    │
                    ├─→ 处理响应
                    │       ├─→ 无工具调用 → 返回 false (结束)
                    │       └─→ 有工具调用 → 执行工具
                    │
                    └─→ 执行工具
                            ├─→ 检查是否需要确认
                            ├─→ 需要确认 → Frontend::request_tool_confirmation()
                            ├─→ 执行工具 (ToolRegistry::execute)
                            ├─→ 发射 ToolCallResult 事件
                            └─→ 返回 true (继续循环)
```

#### 3.2.3 Frontend Trait - 前端抽象

```rust
#[async_trait]
pub trait Frontend: Send + Sync {
    /// Agent 推送事件给前端
    async fn on_event(&self, event: AgentEvent) -> Result<()>;

    /// 请求用户确认工具调用 (阻塞等待)
    async fn request_tool_confirmation(
        &self,
        tool_call: &ToolCallInfo,
    ) -> Result<bool>;

    /// 获取前端消息接收端
    fn event_receiver(&self) -> mpsc::Receiver<FrontendMessage>;
}
```

**设计意图**:
- Agent 不依赖具体前端实现
- 通过 channel 进行异步通信
- 支持同步确认（工具调用）和异步事件

```
crossterm Event
    │
    ├─→ KeyEvent::Char(c)
    │       └─→ input.push(c)

    │

    │

    │
    └─→ KeyEvent::Ctrl+C / Ctrl+D
```

#### 3.3.4 TuiFrontend 实现

```rust
pub struct TuiFrontend {
    app_state: Arc<Mutex<App>>,
    event_tx: mpsc::Sender<FrontendMessage>,
    event_rx: mpsc::Receiver<FrontendMessage>,
    tool_confirmation_tx: mpsc::Sender<bool>,
    tool_confirmation_rx: mpsc::Receiver<bool>,
}

#[async_trait]
impl Frontend for TuiFrontend {
    async fn on_event(&self, event: AgentEvent) -> Result<()> {
        let mut app = self.app_state.lock().await;
        match event {
            AgentEvent::TextDelta(text) => app.append_text(text),
            AgentEvent::ToolCallRequested { tool_call_id, ... } => {
                app.add_tool_call(tool_call_id, ToolCallStatus::Pending)
            }
            AgentEvent::ToolCallResult { tool_call_id, result } => {
                app.update_tool_result(tool_call_id, result)
            }
            AgentEvent::TurnComplete => app.set_status(Status::Ready),
            AgentEvent::Error(e) => app.show_error(e),
            AgentEvent::SkillTriggered { name, ... } => app.show_skill(name),
        }
    }

    async fn request_tool_confirmation(&self, _: &ToolCallInfo) -> Result<bool> {
        self.tool_confirmation_rx.recv().await.ok_or_else(...)
    }

    fn event_receiver(&self) -> mpsc::Receiver<FrontendMessage> {
        self.event_rx.clone()
    }
}
```

## 4. 通信协议

### 4.1 AgentEvent - Agent 到前端

```rust
pub enum AgentEvent {
    /// 流式文本片段
    TextDelta(String),

    /// LLM 请求调用工具
    ToolCallRequested {
        tool_call_id: String,
        name: String,
        arguments: String,
    },

    /// 工具执行结果
    ToolCallResult {
        tool_call_id: String,
        result: ToolResult,
    },

    /// 技能被触发
    SkillTriggered {
        name: String,
        description: String,
    },

    /// 本轮对话结束
    TurnComplete,

    /// 发生错误
    Error(AgentError),
}
```

### 4.2 FrontendMessage - 前端到 Agent

```rust
pub enum FrontendMessage {
    /// 用户新消息
    UserInput(String),

    /// 取消当前操作
    Cancel,

    /// 工具调用确认回复

        id: String,
        approved: bool,
    },
}
```

### 4.3 消息流向

```
[前端]                                  [Agent]
   │                                       │
   │── UserInput ────────────────────────> │
   │                                       ├─ 添加到历史
   │                                       ├─ 上下文管理
   │                                       └─ 调用 LLM
   │                                       │
   │◄─ TextDelta (多次) ────────────────── │
   │◄─ TextDelta (多次) ────────────────── │
   │                                       │
   │◄─ ToolCallRequested ───────────────── │
   │                                       │

   │                                       ├─ 执行工具
   │                                       └─ 添加到历史
   │                                       │
   │◄─ ToolCallResult ──────────────────── │
   │                                       │
   │                                       ├─ 继续 LLM 循环 (如果还有工具)
   │                                       └─ 或结束
   │                                       │
   │◄─ TurnComplete ────────────────────── │
```

## 7. 扩展指南

### 7.1 添加新工具

#### 步骤

1. 在 `crates/robit-agent/src/tool/` 下创建新文件 `my_tool.rs`
2. 实现 `Tool` trait
3. 在 `mod.rs` 中注册
4. 在 `bootstrap.rs` 的 `create_tools_from_config` 中添加

#### 示例: MyTool

```rust
// crates/robit-agent/src/tool/my_tool.rs

use async_trait::async_trait;
use serde_json::json;
use crate::tool::{Tool, ToolContext, ToolResult};

pub struct MyTool;

#[async_trait]
impl Tool for MyTool {
    fn name(&self) -> &str {
        "my_tool"
    }

    fn description(&self) -> &str {
        "Does something useful"
    }

    fn parameters_schema(&self) -> serde_json::Value {
        json!({
            "type": "object",
            "properties": {
                "param1": {
                    "type": "string",
                    "description": "First parameter"
                }
            },
            "required": ["param1"]
        })
    }

    fn requires_confirmation(&self) -> bool {
        true  // 需要用户确认
    }

    async fn execute(
        &self,
        args: serde_json::Value,
        _ctx: &ToolContext,
    ) -> ToolResult {
        let param1 = match args["param1"].as_str() {
            Some(p) => p,
            None => return ToolResult::error("param1 is required"),
        };

        // 执行逻辑
        let result = do_something(param1);

        ToolResult::success(result)
    }
}

// crates/robit-agent/src/tool/mod.rs
mod my_tool;
pub use my_tool::MyTool;

// crates/robit-agent/src/bootstrap.rs
pub fn create_tools_from_config(...) -> ToolRegistry {
    let mut tools = ToolRegistry::new();
    // ...
    if should_enable("my_tool", &enabled_tools) {
        tools.register(MyTool);
    }
    tools
}
```
